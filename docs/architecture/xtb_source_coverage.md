# XTB Source Coverage

Status: approved inventory.
Last updated: 2026-06-17.

Review outcome on 2026-06-18:

- Approved: full XTB statements are the authoritative source baseline.
- Approved: XTB daily PDFs and Gmail-discovered daily PDFs are provisional fast updates.
- Approved: browser automation should be explored before captured endpoint automation, with a separate discovery plan and user approval.

## Purpose

This document inventories implemented XTB source paths for the reconciliation MVP. It separates row parsing from accounting confidence: a source can provide useful evidence before it is safe to use for trusted lifetime P&L, inception history, or broker cash.

Full XTB statements are the authoritative baseline for historical reconciliation because they can contain statement-range trades, cash operations, open/closed position sections, and broker-control evidence. Daily PDFs and Gmail-discovered daily PDFs are provisional fast updates unless reconciled against a full statement for the same period.

## Source Gaps

- No implemented source currently proves broker cash balance against authoritative statement cash totals.
- Full-statement date ranges are parsed indirectly through row dates and private fixture names, but import preview does not expose a first-class coverage range.
- Daily PDF and Gmail coverage is trade-centric; dividends, deposits, withdrawals, cash balances, positions, and corporate actions are missing there.
- Fee/tax/swap support exists for full-statement cash-operation rows, but daily PDF trade fees are only carried inside descriptions.
- Corporate action support is limited to supported split descriptions and correction skipping. Other action types remain unproven.
- Browser automation and captured export endpoint paths are discovery/future work, not current implementation.
